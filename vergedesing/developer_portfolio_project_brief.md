# Project Brief: Developer Portfolio - "Editorial Hazard"

## 1. Project Overview
A high-impact, personal developer portfolio designed with a "brutalist editorial" aesthetic inspired by *The Verge*. The project aims to showcase technical expertise through a lens of high-density information design, using "hazard" color accents (Jelly Mint and Verge Ultraviolet) to create a memorable brand identity.

## 2. Objectives
- **Showcase Technical Depth:** Use a "StoryStream" (git log style) timeline to visualize project history and technical evolution.
- **Brand Identity:** Establish a unique visual voice that stands out from typical "minimalist" portfolios by using bold typography and acid-bright accents.
- **Multi-Mode Versatility:** Provide a seamless experience in both "Canvas Black" (Dark Mode) and "Newsprint White" (Light Mode).
- **Convert Visitors:** Drive engagement through clear, high-contrast Call-to-Actions (CTAs) for project inquiries and social connections.

## 3. Target Audience
- **Tech Recruiters & Engineering Managers:** Looking for clarity, technical stack transparency, and professional history.
- **Design Leads:** Looking for attention to detail, strong typographic hierarchy, and unique aesthetic execution.
- **Potential Clients:** Looking for a bold partner for high-impact web development projects.

## 4. Key Features & User Flows
### Core Sections
1.  **Hero/Identity:** Massive Manuka headline and high-octane mission statement.
2.  **Project Log (StoryStream):** A chronological feed of projects pinned to a dashed timeline rail. Each entry includes:
    *   Timestamp/Date
    *   Project Title & Description
    *   Technical Tags (Pills)
    *   Visual Reference (Bordered Imagery)
3.  **Tech Stack Inventory:** A categorical breakdown of languages, frameworks, and infrastructure tools.
4.  **Professional History:** A detailed log of past roles using editorial-style cards.
5.  **Connection Terminal (CTA):** A high-impact contact section with bold inputs and "Hazard" fills.

### Navigation
*   **Sticky Top Bar:** Providing quick access to Projects, Experience, Skills, and Archive.
*   **Connect CTA:** Persistent primary action button in the header.

## 5. Visual Language (Editorial Hazard)
*   **Typography:** 
    *   **Manuka:** Used for massive headlines and wordmarks (The "Shout").
    *   **PolySans:** The primary workhorse for body copy and headings.
    *   **PolySans Mono:** Used for timestamps, buttons, and technical labels (The "Log").
*   **Color Palette:**
    *   **Primary:** Jelly Mint (`#3CFFD0`) for high-priority actions and active states.
    *   **Secondary:** Verge Ultraviolet (`#5200FF`) for accents and secondary signals.
    *   **Surfaces:** Canvas Black (`#131313`) and Newsprint White (`#F9F9F9`).
*   **Elevation:** No shadows. Depth is communicated via 1px hazard borders and saturated color fills.

## 6. Technical Requirements
*   **Responsive Design:** Optimized for Desktop with a clear strategy for mobile adaptation.
*   **Theming Engine:** Support for Light/Dark mode switching while maintaining brand consistency.
*   **Performance:** Fast-loading images with 1px hairline frames to ensure no layout shift.

## 7. Future Considerations
*   **Interactive Log:** Expanding StoryStream to include small technical blog posts or "logs" alongside projects.
*   **Project Detail Views:** Dedicated long-form "Editorial Case Study" pages for major work.
*   **Figma Export:** Ready for transition into high-fidelity prototype tools.
