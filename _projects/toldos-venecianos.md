## De prompt a producción con Claude Design

Proyecto personal de investigación para explorar un flujo de trabajo nuevo: usar **Claude Design** para generar la propuesta visual completa de una landing page y después implementarla en **React 18** sin alterar el diseño resultante.

Como caso de uso elegí una landing ficticia de toldos venecianos — un proyecto sin cliente real que me permitía trabajar con libertad total y con un tipo de contenido (landing comercial con secciones, CTA, galería) lo suficientemente representativo.

### El flujo

| Fase | Herramienta | Resultado |
|---|---|---|
| **Diseño visual** | Claude Design | Propuesta de layout, paleta, tipografía y componentes |
| **Implementación** | React 18 + Vite | Landing funcional y responsiva |
| **Deploy** | GitHub Pages | Publicación automática desde el repositorio |

### Qué quería probar

- ¿Qué tan usable es la propuesta de Claude Design como especificación de diseño real?
- ¿Se puede llevar directamente a código sin reinterpretar o rediseñar?
- ¿El resultado final es lo suficientemente coherente y pulido para producción?

### Aprendizajes

Claude Design genera propuestas visuales con suficiente detalle (colores, tipografía, estructura de secciones) para pasarlas directamente a código. El mayor valor está en la velocidad — pasar de idea a diseño funcional en horas en lugar de días. El resultado es coherente y visualmente limpio, aunque requiere juicio propio para ajustar detalles de UX que la IA no siempre resuelve de forma óptima.
